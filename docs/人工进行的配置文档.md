# 人工进行的配置文档

`blog-global` 的翻译逻辑无需人工介入，但以下几项必须由你手动完成一次。

## 1. AI Provider API Key

二选一：

- **硅基流动 (SiliconFlow)**：https://cloud.siliconflow.cn → 账户 → API 密钥。`base_url` 固定为 `https://api.siliconflow.cn/v1`，模型名需带组织前缀，如 `deepseek-ai/DeepSeek-V3`。
- **OpenRouter**：https://openrouter.ai/keys 。`base_url` 固定为 `https://openrouter.ai/api/v1`。

模型建议：`deepseek-ai/DeepSeek-V3`（已实测翻译质量稳定，能正确返回 JSON、保留代码块/链接/图片路径）。7B 级小模型（如 Qwen2.5-7B-Instruct）实测容易返回格式错乱的 JSON 导致翻译失败，不建议用于生产。

## 2. GitHub Token

用于自动创建 PR，二选一：

### 方案 A：默认 GITHUB_TOKEN（零配置，推荐先用这个）

在 GitHub Actions 里什么都不用配，`action.yml` 的 `github_token` 输入默认取 `${{ github.token }}`。

限制：由它创建的 PR 不会触发仓库里其他的 workflow（比如你的站点部署 workflow 不会被自动触发，需要你手动合并后再触发，或换方案 B/C）。

### 方案 B：Personal Access Token（本地 CLI 必须用这个）

1. GitHub → Settings → Developer settings → Personal access tokens → Fine-grained tokens（推荐）或 Tokens (classic)。
2. Fine-grained：Repository access 选中你的博客仓库；Permissions 勾 `Contents: Read and write`、`Pull requests: Read and write`。
   Classic：直接勾 `repo`。
3. 本地 CLI 使用：
   - PowerShell: `$env:GITHUB_TOKEN="your_pat"`
   - Bash: `export GITHUB_TOKEN="your_pat"`
   - 或运行时用 `--token your_pat` 传入。
4. 若要在 Action 里用这个 PAT 而不是默认 GITHUB_TOKEN，存成仓库 Secret（如 `PAT_TOKEN`），并在 workflow 里显式传给 `github_token` 输入（见《快速启动说明文档》）。

### 方案 C：GitHub App（多仓库/机器人身份场景再考虑，MVP 阶段可跳过）

1. Settings → Developer settings → GitHub Apps → New GitHub App，Webhook 的 `Active` 取消勾选。
2. 权限：`Contents: Read & Write`、`Pull requests: Read & Write`。
3. 生成 Private Key，记下 App ID，安装到目标仓库。
4. 存 `APP_PRIVATE_KEY`（Secret，填 `.pem` 全文）和 `APP_CLIENT_ID`（Variable）。

## 3. 博客仓库准备

- 必须是 Jekyll 博客（有 `_config.yml` 和 `_posts/` 目录）。目前仅 Jekyll 经过完整测试。
- 仓库需要有 `main`（或你指定的）分支，且已有至少一次提交（PR 需要基于已存在的 base 分支创建）。
- 建议 `.github/workflows/` 触发的 push 权限包含 `contents: write` + `pull-requests: write`（见快速启动文档的 workflow 示例）。

## 4. 可选配置文件（放在博客仓库根目录）

### `.blog-global.yml`（省得每次 CLI 交互都重新输一遍）

```yaml
provider: siliconflow
model: deepseek-ai/DeepSeek-V3
base_lang: zh-CN
target_langs:
  - en
  - ja
```

API Key 不要写进这个文件，用环境变量 `BLOG_GLOBAL_API_KEY` 或 `--api-key` 传。

### `.blog-global-ignore`（不想翻译的文章）

```
_posts/2024-01-01-draft.md
_posts/private-*.md
```

## 5. 若你要发布/分发这个 Action 或 CLI（非必须，仅供参考）

- `action.yml` 通过 `uses: 你的用户名/blog-global@版本号` 引用，GitHub 直接拉取该 tag 下的 `dist/index.js` 运行，**必须把 `dist/` 目录提交进 git**（仓库 `.gitignore` 已经放开了 `dist/`，正常 `git add` 就会带上）。
- `npm install -g blog-global` 依赖 `package.json` 里 `bin` 指向的 `dist/cli.js`，同样要求 `dist/` 已构建并被提交（本地开发用 `npm publish` 会在发布前自动构建，无需额外操作；只有走 git 仓库直接分发时才需要手动确认 `dist/` 是最新的）。
