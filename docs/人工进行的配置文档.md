# 人工进行的配置文档

为了使 `blog-global` 能够正常运行并自动创建 Pull Request，您需要完成以下人工配置：

## 1. 获取 AI Provider API Key

您需要注册并获取以下任一 AI 平台的 API Key：

- **OpenRouter**: 访问 [https://openrouter.ai](https://openrouter.ai) 注册并生成 API Key。
- **硅基流动 (SiliconFlow)**: 访问 [https://siliconflow.cn](https://siliconflow.cn) 注册并生成 API Key。

---

## 2. GitHub 权限与 Token 配置

### 方案 A：使用 GitHub 个人访问令牌 (Personal Access Token - PAT)

如果您打算在本地 CLI 运行，或在 GitHub Actions 中使用更高级的权限，需要生成一个 PAT：

1. 访问 GitHub 个人设置：**Settings -> Developer settings -> Personal access tokens -> Tokens (classic)**。
2. 点击 **Generate new token (classic)**。
3. 勾选 **repo** 权限范围（包含 `repo:status`, `public_repo`, `repo_deployment` 等）。
4. 生成并复制 Token。
5. **本地运行**：将该 Token 设置为环境变量：
   - Windows (PowerShell): `$env:GITHUB_TOKEN="your_pat_token"`
   - Linux/macOS: `export GITHUB_TOKEN="your_pat_token"`
6. **Actions 运行**：将该 Token 存入您博客仓库的 **Settings -> Secrets and variables -> Actions -> New repository secret**，命名为 `BLOG_GLOBAL_GITHUB_TOKEN`。

### 方案 B：使用 GitHub App (推荐用于 Actions)

如果您希望以独立的 Bot 身份运行并自动创建 PR：

1. 访问 GitHub 个人设置：**Settings -> Developer settings -> GitHub Apps -> New GitHub App**。
2. 填写基本信息：
   - **GitHub App name**: `blog-global-bot`
   - **Homepage URL**: 您的博客地址或任意有效 URL。
3. **Webhook 配置**：
   - **取消勾选** `Active` (即不启用 Webhook，本项目不需要接收 Webhook 回调)。
4. 配置权限 (Repository permissions)：
   - **Contents**: `Read & Write`
   - **Pull requests**: `Read & Write`
5. 创建 App 后，生成并下载 **Private Key** (私钥 `.pem` 文件)，并记录 **App ID (Client ID)**。
6. 在您的博客仓库中安装该 GitHub App：**Install App -> Only select repositories -> 选择您的博客仓库**。
7. 在博客仓库的 **Settings -> Secrets and variables -> Actions** 中添加以下配置：
   - **Repository Secret**: `APP_PRIVATE_KEY` (填入 `.pem` 文件的完整文本内容)。
   - **Repository Variable**: `APP_CLIENT_ID` (填入 App ID)。

---

## 3. 博客仓库配置

在您的博客仓库根目录下，您可以选择性创建以下两个配置文件：

### 3.1 基础配置文件 `.blog-global.yml`

避免每次运行 CLI 时重复输入配置（支持指定 `base_branch`）：

```yaml
provider: "siliconflow"
model: "deepseek-ai/DeepSeek-V3"
base_lang: "zh-CN"
target_langs:
  - "en"
  - "ja"
base_branch: "main"
```

### 3.2 忽略翻译文件 `.blog-global-ignore`

列在其中的文章文件不会被翻译（支持 `.gitignore` 语法）：

```
# 忽略草稿或测试文章
_posts/2024-01-01-draft.md
_posts/test-*.md
```
