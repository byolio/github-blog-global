# 快速启动说明文档

本指南将帮助您在本地或 GitHub Actions 中快速启动并运行 `blog-global` 项目。

## 1. 本地快速启动 (CLI 模式)

### 1.1 安装依赖并构建

在项目根目录下执行以下命令：

```bash
# 安装依赖
npm install --ignore-scripts

# 编译 TypeScript 并打包 CLI
npm run build
npm run package:cli
```

### 1.2 交互式运行

在您的博客仓库根目录下，直接运行编译后的 CLI 脚本：

```bash
node F:/AICode/blog-global/dist/cli.js
```

CLI 将引导您输入（**API Key 和 Token 等输入框在控制台显示为明文，方便您粘贴和检查，无需担心被掩码**）：

1. 选择 AI Provider (OpenRouter / SiliconFlow / Custom)
2. 输入 API Key
3. 选择翻译模型
4. 输入源语言 (如 `zh-CN`)
5. 选择目标语言 (如 `en`, `ja`)
6. 输入 PR 目标基准分支 (如 `main`, `master`)
7. 确认是否自动创建 Pull Request

### 1.3 命令行参数运行 (非交互模式)

当本地配置了环境变量并提供参数时，CLI 将跳过交互提示直接运行：

```bash
# 设置环境变量
$env:BLOG_GLOBAL_API_KEY="your_ai_api_key"
$env:GITHUB_TOKEN="your_github_pat"

# 运行 CLI (支持 --base-branch 选项)
node F:/AICode/blog-global/dist/cli.js `
  --provider openrouter `
  --model google/gemini-2.5-flash `
  --base-lang zh-CN `
  --target-langs en,ja `
  --base-branch main `
  --create-pr
```

### 1.4 进度实时展示与极佳的 PR 创建重试环

- **进度实时输出**：翻译运行时控制台将以如下格式输出当前翻译文件数、总文件数、完成 AI 任务数以及使用的 AI 模型，方便您掌控全局进度。
  ```text
  [Progress] 1/46 files translated (AI tasks: 1/46 using deepseek-ai/DeepSeek-V4-Pro)
  ```
- **PR 创建重试环**：AI 翻译完成并写盘后，若在创建分支、推送代码、创建 PR 任何一步发生错误（如 Token 无效、目标分支不存在等），**程序绝不崩溃退出，已翻译的文章不会丢失**。控制台将立刻打印错误，并弹出交互式重试环：
  ```text
  ? How would you like to proceed? (Use arrow keys)
  ❯ Change Base Branch and Retry (修改 PR 目标基准分支并重新创建)
    Change GitHub Token and Retry (重新输入 Token 密钥并重新创建)
    Skip Pull Request creation (跳过 PR 创建，将翻译好的文件保留在本地)
  ```

## 2. GitHub Actions 快速启动 (Action 模式)

### 2.1 打包 Action

在发布到 GitHub 之前，必须将 Action 打包为单文件：

```bash
npm run package
```

这将在 `dist/index.js` 中生成打包后的代码。请确保将 `dist/index.js` 和 `action.yml` 提交并推送到您的 GitHub 仓库。

### 2.2 在博客仓库中配置 Workflow

在您的博客仓库中创建 `.github/workflows/translate.yml` 文件：

#### 简单配置 (使用 GITHUB_TOKEN)

```yaml
name: Blog Global Translate
on:
  push:
    branches: [main]
    paths: ['_posts/**']

permissions:
  contents: write
  pull-requests: write

jobs:
  translate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0 # 必须获取完整历史以支持增量检测

      - uses: your-username/blog-global@main
        with:
          ai_provider: openrouter
          ai_api_key: ${{ secrets.AI_API_KEY }}
          ai_model: google/gemini-2.5-flash
          base_lang: zh-CN
          target_langs: en,ja
```

#### 推荐配置 (使用 GitHub App)

```yaml
name: Blog Global Translate
on:
  push:
    branches: [main]
    paths: ['_posts/**']

jobs:
  translate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/create-github-app-token@v3
        id: app-token
        with:
          client-id: ${{ vars.APP_CLIENT_ID }}
          private-key: ${{ secrets.APP_PRIVATE_KEY }}
          permission-contents: write
          permission-pull-requests: write

      - uses: actions/checkout@v4
        with:
          token: ${{ steps.app-token.outputs.token }}
          fetch-depth: 0

      - uses: your-username/blog-global@main
        with:
          ai_provider: openrouter
          ai_api_key: ${{ secrets.AI_API_KEY }}
          ai_model: google/gemini-2.5-flash
          base_lang: zh-CN
          target_langs: en,ja
          github_token: ${{ steps.app-token.outputs.token }}
```
